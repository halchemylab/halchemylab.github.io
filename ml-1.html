Restaurant Rating Prediction

Machine Learning and Streamlit

Project Overview

This project leverages exploratory data analysis (EDA), data cleaning, and machine learning techniques to create actionable insights for a restaurant dataset. It culminates in an interactive Streamlit app, showcasing predictions and data exploration through a user-friendly interface. By harnessing the best-performing machine learning model, GridSearchCV-tuned Random Forest Regressor (GridRFR), the app provides reliable predictions while demonstrating the end-to-end data science workflow.

Key Steps and Methodology

1. Exploratory Data Analysis (EDA)

The project began by delving into the dataset to uncover key insights:

  • Top Cuisines: North Indian cuisine dominated the dataset, with over 53,000 votes, followed by Chinese and Mughlai options.
  • Rating Analysis: Most restaurants received an "Average" or "Good" rating, while only a small fraction excelled or performed poorly.

These findings shaped the strategy for feature selection and model development.

2. Data Cleaning

The raw dataset included columns such as Restaurant Name, Country Code, City, and Cuisines. To prepare the data for modeling:

  • Missing Values: Addressed incomplete entries by imputing or dropping records based on their importance.
  • Encoding: Transformed categorical variables like Cuisines and Rating text into numerical formats using one-hot and label encoding.
  • Outliers: Detected and handled outliers in numerical fields such as Average Cost for two and Votes.
  • Scaling: Standardized features using a scaler to ensure consistent model performance.

3. Feature Engineering

Key transformations and feature creation included:

  • Geospatial Features: Leveraged Longitude and Latitude for location clustering.
  • Derived Metrics: Created composite metrics such as Cost per Vote to add more granularity to the analysis.

4. Machine Learning

The data was split into training and testing sets. Several machine learning models were evaluated, including:

  • Linear Regression
  • Decision Trees
  • Random Forest Regressor
  • GridSearchCV-tuned Random Forest Regressor (GridRFR)

The GridRFR outperformed all models with the lowest RMSE and highest RÂ² score, making it the backbone of the predictions. This model was saved as a .pkl file for deployment.

5. Streamlit App Development

To ensure accessibility, an intuitive Streamlit app was developed. Features included:

  • Data Exploration: Users could visualize restaurant data trends, filter by cuisines, and inspect geospatial distributions.
  • Predictive Insights: Users could input restaurant features (e.g., Cuisines, Cost, Votes) to predict outcomes like expected ratings or customer engagement.
  • Model Deployment: The app integrated the saved scaler and model .pkl files, allowing real-time predictions.

Results and Impact

  • Actionable Insights: Provided restaurant businesses with tools to understand and optimize their offerings.
  • Predictive Power: Delivered reliable forecasts to support data-driven decisions.
  • User Experience: The Streamlit app's interactive interface made it simple for non-technical users to explore and apply the insights.

Technical Stack

  • Data Analysis: Python (Pandas, NumPy, Matplotlib, Seaborn)
  • Machine Learning: Scikit-learn (Random Forest, GridSearchCV)
  • App Deployment: Streamlit
  • File Management: Pickle for model and scaler storage
